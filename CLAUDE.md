## Workflow Rules
- **Always ask before committing.** Never run `git commit` or `git push` without explicit user confirmation.
- Keep changes minimal and focused — no over-engineering, no unsolicited refactors.
- Do not add comments, docstrings, or type annotations to code you didn't change.
- Prefer editing existing files over creating new ones.

## Code Style
- Use functional React components with hooks.
- Consistent naming: camelCase for variables/functions, PascalCase for components.
- No unused imports or dead code.
